Design low-latency APIs with channel authorization, resync logic, and scalable event fanout. ← Back · Practice

What real-time APIs solve

Real-time APIs push updates immediately instead of waiting for client polling intervals. They are common in dashboards, chats, operations consoles, multiplayer apps, and collaborative editing.

Key design decisions

  • Transport: WebSocket, SSE, MQTT, or mixed
  • Fanout topology: single node vs distributed pub/sub
  • Ordering and sequence guarantees
  • Reconnect + state catch-up behavior
  • Authorization per channel/topic

Operational metrics

  • Connection count and churn
  • p95 end-to-end latency
  • Dropped message rate
  • Reconnect success rate
